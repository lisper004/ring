## Preamble

The REPLOS and RING projects are free software, built on the principles of user freedom, transparency, and community collaboration. As part of this commitment, we establish a clear policy regarding the use of Artificial Intelligence systems for contributing to, interacting with, or generating content for these projects.

We recognize that AI can be a tool, but we reject the use of proprietary, centralized, and user-hostile "AI" services (such as ChatGPT, GitHub Copilot, Google Gemini, Claude, and similar chatbots). These services:
- Operate under centralized corporate control
- Are trained on data without proper consent
- Often generate incorrect, inefficient, or low-quality code
- Undermine the spirit of free software by locking users into proprietary dependencies

Therefore, this policy defines the only acceptable conditions for using AI in relation to the REPLOS and RING projects.

## 0. Definitions

**"Proprietary AI"** : Any AI system that is not distributed under a license approved by the Free Software Foundation (FSF) or the Open Source Initiative (OSI), or that requires payment, proprietary software, or centralized servers to function.

**"Free AI"** : An AI system that is distributed under a free software license, can be run locally on user-controlled hardware, does not require external servers, and does not impose restrictions on use or modification.

**"Contribution"** : Any code, documentation, bug report, suggestion, or other material submitted to the REPLOS or RING projects (including pull requests, issues, discussions, and direct commits).

**"Chatbot"** : Any AI system designed to generate natural language or code responses in a conversational interface, including but not limited to large language models (LLMs).

## 1. Prohibited Activities

The following activities are strictly prohibited when interacting with the REPLOS or RING projects:

**a) Using Proprietary AI for contributions.** You may not use ChatGPT, Copilot, Gemini, Claude, or any other proprietary AI service to generate:
   - Code submitted as a contribution
   - Documentation (including comments, README files, or help text)
   - Bug reports or feature requests (unless manually rewritten without AI)
   - Responses in issues, discussions, or pull requests

**b) Submitting AI-generated garbage.** You may not submit code or text generated by any AI (including Free AI) that is:
   - Non-functional or contains logic errors
   - Inefficient or unnecessarily complex
   - Plagiarized from copyrighted sources
   - Not reviewed and tested by a human before submission

**c) Using chatbots for interactive development.** You may not use a chatbot to:
   - Debug or analyze REPLOS/RING code on your behalf
   - Generate commit messages or pull request descriptions
   - Create release notes or change logs

**d) Training proprietary AI on project data.** You may not use any code, documentation, or other data from REPLOS or RING to train a Proprietary AI system without explicit written permission from the copyright holder.

## 2. Permitted Activities

The following uses of AI are permitted, as long as they respect the spirit of free software:

**a) Using Free AI locally.** You may use Free AI systems (such as locally-run instances of Llama, Mistral, BLOOM, or similar) if:
   - The AI runs entirely on your own hardware
   - The AI software is distributed under a free software license
   - You do not rely on external servers or proprietary APIs
   - You manually verify and test all output before submitting it

**b) Using AI for non-contributory purposes.** You may use any AI for:
   - Personal learning or experimentation that does not result in contributions
   - Generating ideas or pseudocode that you rewrite completely from scratch
   - Translating existing documentation (as long as you review the result)

**c) Using AI for meta-project tasks.** You may use Free AI for:
   - Code review assistance (as long as the reviewer is a human)
   - Security analysis (with all findings manually verified)
   - Build system optimization (with all changes manually reviewed)

## 3. Disclosure and Attribution Requirements

If you use any AI (including Free AI) to help prepare a contribution, you **MUST** disclose this in the contribution:

- For pull requests: Add a comment or section titled "AI Disclosure" stating:
  - Which AI system was used (name, version, and whether it is Free or Proprietary)
  - Which parts of the contribution were AI-generated
  - The steps taken to verify correctness and efficiency

- For issues or discussions: If you use AI to help formulate the message, state this clearly at the beginning.

Failure to disclose AI use is considered a violation of this policy.

## 4. Enforcement

Violations of this policy will be handled as follows:

| Violation | Consequence |
|-----------|-------------|
| First offense (minor, e.g., undisclosed Free AI use) | Warning and request to amend contribution |
| Second offense or first offense with Proprietary AI | Rejection of contribution and public notice |
| Repeated violations or intentional abuse | Permanent ban from the project repositories |

Project maintainers have final authority in interpreting and enforcing this policy.
